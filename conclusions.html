Visual Patterns with Code

Conclusions

Specificity is the enemy of complexity.

Location based styles should be used sparingly.

Look at your site, catalog it. Cut out the UI pieces. Then look for patterns.

Code the patterns - semantic HTML.

Abstract the CSS - reuse what you can.

Use location based CSS to build reusable modular components.

One more thought. How old is your code?

Cascading Style Sheets... We like the cascade, let's not abuse it.
